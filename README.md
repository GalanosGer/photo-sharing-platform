# 📷 k29photo

A full-stack Flickr-style photo sharing platform developed as an individual university project using **Flask**, **Python**, and **PostgreSQL**. The project demonstrates relational database design, backend web development, and advanced SQL through the implementation of a complete social photo-sharing application.

---

## Features

- 👤 User registration and authentication
- 📸 Photo uploads
- 📁 Album management
- 🏷️ Tagging system
- ❤️ Like photos
- 💬 Comment on photos
- 🤝 Friendship management
- 👥 Friend recommendation system
- 🔍 Advanced AND tag search
- 🔎 Full-text comment search
- 🏆 User contribution ranking
- 🎯 "You May Also Like" recommendation algorithm based on tag similarity

## Database Design

The database was designed entirely from scratch following relational database design principles.

Key concepts include:

- Entity-Relationship (E/R) Modeling
- Schema Normalization
- Foreign Keys
- Primary Keys
- SQL Constraints
- Triggers
- Complex SQL Queries

The database supports entities including:

- Users
- Albums
- Photos
- Tags
- Comments
- Likes
- Friendships

## Project Highlights

- Designed a complete normalized relational database schema.
- Developed the full Flask backend independently.
- Implemented advanced SQL functionality including ranking, recommendation, and search algorithms.

## Installation

Clone the repository:

```bash
git clone https://github.com/GalanosGer/k29photo.git
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the application:

```bash
python app.py
```
